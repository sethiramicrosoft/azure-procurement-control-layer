# Internalization Plan (Microsoft)

## Goal

Package APCL as a repeatable field accelerator that helps customers implement procurement-aware Azure governance.

## Positioning

APCL is not a replacement for customer procurement systems. It is the Azure control-plane bridge between procurement intent and technical consumption.

## Field motion

1. Discovery workshop (procurement + platform + finance).
2. Control matrix definition (preventive/detective/corrective).
3. Pilot in one governed scope.
4. Evidence capture (bypass reduction, attribution quality, approval SLA).
5. Scale-out blueprint across business units.

## Success criteria

- Reduction in unapproved spend paths.
- Increase in tagged, attributable spend.
- Lower invoice surprises.
- Faster approvals for pre-approved patterns.

## Packaging assets

- Reference architecture.
- Policy packs by profile.
- Deployment templates.
- Customer workshop kit.
- KPI dashboard starter.
